# 架构 V2 五项任务实施交接

更新时间：2026-07-23

## 完成状态

1. 精简存储：完成。
2. 统一资产与决策关联：完成。
3. 资讯聚类与确认后转证据：完成。

5. 到期结算与自动校准门槛：完成。

## 用户可见变化

- “数据、导入与恢复”新增存储健康和“整理可回收空间”；
- 概率研报不再要求用户填写模型编号、样本数和 Brier 分；
- 选择标的后自动导入已确认的持仓证据；
- 样本不足时显示“实验贝叶斯”，不会冒充正式概率；
- 实验概率允许到期结算，结算后自动进入校准样本；
- 用户确认 AI 持仓影响后，系统自动创建事实证据和影响分析证据。

## 关键接口

```text
GET  /api/storage/status
POST /api/storage/compact                 { "confirmed": true }

GET  /api/probability-calibration?horizon=5d
POST /api/probability-reports/preview
POST /api/probability-reports
POST /api/probability-reports/{id}/resolve

POST /api/information-events/{id}/holding-impact-confirmations
```

## 数据实体

- `assets`：统一资产主档；
- `trade.decisionContext`：成交时冻结的计划、研究、证据与账户上下文；
- `informationEventClusters`：跨来源事件簇；
- `informationEvidencePromotions`：确认结果转化为研究证据的血缘记录；
- `probabilityReports[].bayesian`：先验、似然比、逐步后验和防重复计权记录；
- `forecastResolutions`：到期结果、冻结概率和 Brier 分。

## 后续边界

- 不得把实验概率、影响分或纪律分描述为确定涨跌概率；
- 不得删除事件簇中的各来源原始记录；
- 不得让未确认的 AI 持仓影响自动进入研究证据；
- 不得手工填写或伪造校准样本数；
- 将来可继续把 `/api/store` 的兼容聚合读取拆成模块接口和游标分页，但这不影响本次五项闭环运行。
