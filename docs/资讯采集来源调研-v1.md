# 资讯采集来源调研 v1

调研日期：2026-07-22

目标不是“抓得越多越好”，而是获得可持续、可核验、可回到原文的事实流。任何来源在进入自动采集前，必须同时满足：访问方式获得授权、原文链接可保存、发布时间可识别、增量游标可实现、失败不会污染账本。

## 已核验的一手能力

### 中国政府采购网

中国政府采购网设有正式的“数据标准及规范”专区。页面显示《中国政府采购网数据接口规范》当前版本为 v1.2（2026-03-16）。这是采购公告、采购意向和中标结果最值得优先研究的官方接入方向，但具体接口地址、认证和使用范围仍需按规范申请与核对，不能把网页列表抓取当成正式接口。

- 官方入口：https://www.ccgp.gov.cn/sjbzjgf/

### 沪深交易所与法定披露

上海证券交易所提供公众公告检索，也发布面向市场参与者的技术接口规范；两者授权对象不同。主程序不直接复用网页内部请求。后续适配器应优先采用明确允许程序访问的公告数据服务、正式订阅或经核验的开放接口，并对上海、深圳、北京市场分别实现。

- 上交所公告检索：https://www.sse.com.cn/assortment/stock/list/info/announcement/index.shtml
- 上交所技术接口专区：https://www.sse.com.cn/services/tradingtech/data/
- 深交所官网：https://www.szse.cn/
- 巨潮资讯网：http://www.cninfo.com.cn/

### 官方 RSS / Atom

RSS/Atom 是当前国内官方来源的首选通道：不需要令牌、原文链接和发布时间可追踪，也适合增量轮询。实际联调已经确认国家统计局两个 XML 源可稳定读取。外交部和国防科工局虽然仍有官方 RSS 说明页，但其 XML 地址在 2026-07-22 联调时返回失效页或无可识别条目，因此不自动启用。其他部委和地方站点只有在明确公布订阅地址且实际试采集成功时才加入，不猜测内部接口。

- 国家统计局 RSS 说明：https://www.stats.gov.cn/wzgl/rss/202302/t20230217_1912859.html
- 国家统计局最新发布：https://www.stats.gov.cn/sj/zxfb/rss.xml
- 国家统计局数据解读：https://www.stats.gov.cn/sj/sjjd/rss.xml
- 外交部领事服务 RSS 说明（当前 XML 联调未通过）：https://cs.mfa.gov.cn/rss/
- 国防科工局 RSS 入口（当前 XML 联调未通过）：https://www.sastind.gov.cn/n10118180/index.html

### SEC EDGAR

SEC 的 `data.sec.gov` 提供无需认证和 API Key 的公开 JSON 接口，可按公司 CIK 获取实时更新的申报列表。程序要求用户填写包含联系邮箱的 User-Agent，以遵守 SEC 的公平访问规范；这不是注册或付费。

- 官方 API 说明：https://www.sec.gov/search-filings/edgar-application-programming-interfaces
- 官方公平访问规范：https://www.sec.gov/about/developer-resources

### Federal Register

FederalRegister.gov 提供无需 API Key 的官方 JSON API，可按关键词和联邦机构查询规则、拟议规则、通知和总统文件。它用于国际政策与产业环境线索，不替代公司层面的传导核验。

- 官方开发者入口：https://www.federalregister.gov/developers/documentation/api/v1

### X 官方接口：付费并停用

X 当前按读取资源计费，不再作为本项目的默认或可启用来源。旧配置会在迁移时自动停用，后端也会拒绝运行；保留旧适配器代码只为数据兼容，不代表推荐接入。

- 官方定价：https://docs.x.com/x-api/getting-started/pricing

### 政府与地方公共数据

国家政策要求公共数据在安全和隐私前提下依法有序开放，并提高机器可读性。地方开放平台的接口结构和授权方式不同，适合通过独立适配器逐一接入，不适合写一个脆弱的通用网页爬虫。

- 公共数据资源开发利用意见：https://www.mee.gov.cn/zcwj/zyygwj/202410/t20241010_1087779.shtml

## 当前落地

1. `RSS / Atom` 适配器，用于政府、监管和公司官方 XML 订阅；
2. 标准 `JSON Feed` 适配器，用于官方或自建结构化源；
3. `SEC EDGAR submissions` 适配器，按 CIK 获取公司法定申报；
4. `Federal Register` 适配器，按关键词或机构获取美国联邦政策；
5. 零费用硬约束：付费 X 来源不能启用或执行，旧配置自动停用；
6. 50–75 分钟默认随机采集窗口，可按来源调整；
7. 每个来源保存游标、上次运行、下次运行、最近错误；
8. 每个采集事件保存来源 ID、外部编号、原始内容指纹和原文链接；
9. 相同外部编号跨轮次幂等去重，失败运行进入审计但不产生资讯事件；
10. 来源适配器和影响评估引擎分离，其他 AI 代理可通过 HTTP API 或项目 Skill 使用。
11. RSS/JSON 首次接入默认只读取最近 30 天且最多 50 条；失败不推进成功检查点，更换地址会自动重置检查点。

## 暂不接入主程序的来源

X、雪球、财联社快讯、所谓“闪电资讯”以及其他付费、需要模拟登录、复用网页私有接口或绕过访问限制的来源，目前只保留为人工线索。只有在找到明确免费的正式开放接口或书面许可后，才增加独立适配器。转载与社交内容即使未来接入，也只能作为线索，不自动提升为一手事实。

## 下一批适配器优先级

1. `ccgp_v1_2`：按正式规范申请并验证采购意向、招标和中标增量接口；
2. `exchange_disclosure`：核对授权后接入沪深北交易所或法定披露数据服务；
3. `government_open_data`：为有正式 JSON/API 的中央和地方政策平台建立配置化映射；
4. `company_ir_feed`：公司投资者关系或公告的官方结构化源；
5. 新闻和社交聚合：仅作低等级线索层，并强制寻找原始来源。
