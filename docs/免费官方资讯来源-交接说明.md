# 免费官方资讯来源交接说明

> 本文记录的是仅接入国家统计局 RSS 的早期阶段。NewsNow 聚合来源和本地原文存储已经在后续阶段完成，请以 `docs/NewsNow与原文存储-交接说明.md` 为准。

交接日期：2026-07-22

## 一、已经替你完成的事项

目前应用已经切换为“零费用官方来源模式”，后台服务也已经重启到新版本。你不需要申请 X 令牌，也不需要购买任何接口套餐。

已经配置、试采集并启用两个中国官方来源：

| 来源 | 用途 | 当前状态 | 已保留资讯 |
| --- | --- | --- | ---: |
| 国家统计局最新发布 | 宏观经济与官方统计发布 | 已启用 | 50 条 |
| 国家统计局数据解读 | 官方统计口径和数据解释 | 已启用 | 50 条 |

系统会在每次成功采集后，随机安排下一次运行时间，默认间隔为 50–75 分钟。失败不会推进成功检查点，也不会把错误内容写成资讯。

首次联调时国家统计局 RSS 返回了大量历史记录。已经安全清理 900 条多余历史记录，每个统计来源仅保留最新 50 条。清理前应用自动生成了数据库备份，如有需要可以恢复。

## 二、你现在只需要怎么做

### 第一步：刷新页面

打开交易纪律助手。如果页面之前一直开着，请按一次 `Ctrl + F5` 强制刷新。

### 第二步：进入资讯影响中心

在左侧菜单进入“资讯影响中心”。顶部“自动采集来源”区域应当看到两个来源，并显示：

- 自动运行；
- 免费官方源；
- 最近运行没有红色错误；
- 下一次运行时间。

### 第三步：查看资讯，不要把资讯直接当成买卖信号

采集进来的内容会出现在“待处理资讯”。先看标题、官方来源和原文，再决定是否值得做公司影响评估。

资讯的重要性和某只股票是否真正受益是两件事。只有完成公司九段传导链核验后，系统才会形成公司影响分；缺少招投标资格、历史项目、产品匹配或收入重要性证据时，不应直接得出“公司受益”的结论。

## 三、以后自己增加官方 RSS 时怎么填

进入“资讯影响中心”→“配置来源”，逐项填写：

1. 来源名称：填写官方机构和栏目，例如“某省工信厅通知公告”。
2. 免费适配器：选择“官方 RSS / Atom”。
3. 证据来源类别：政府政策选择“政府 / 监管政策”；官方统计选择“官方统计”。
4. RSS / Atom 地址：必须是官网明确公布的 XML 地址，通常以 `.xml` 结尾。
5. 最短间隔：填写 `50`。
6. 最长间隔：填写 `75`。
7. 初次配置建议先不要勾选自动采集，保存后点击一次“立即采集”。
8. 看到采集成功且原文链接正确后，再点击“启用”。

不要把普通新闻列表网页地址填进 RSS 地址。只有官网明确写着“RSS订阅”并给出 XML 地址时才使用。

## 四、哪些东西不要配置

- 不需要 X Bearer Token；X 当前属于付费来源，系统已经阻止。
- 不要购买所谓“资讯聚合接口”，目前没有这个必要。
- 不要填写雪球、财联社等网站的登录 Cookie。
- 不要使用来源不明的网页私有接口。
- 不要因为采集到利好标题就直接交易，必须继续核验公司传导链和技术条件。
- 当前只做中国股票，不配置 SEC、Federal Register 等美国来源。

## 五、出现问题时怎么看

如果某个来源出现红色错误：

1. 先点击“停用”，避免连续失败；
2. 打开官方 RSS 说明页，确认 XML 地址有没有变化；
3. 不要反复填写账号或令牌；
4. 把错误截图或原文地址发给开发代理，由代理修改适配器或替换来源。

如果页面没有看到新功能，先按 `Ctrl + F5`。后台服务当前运行在 `http://127.0.0.1:3768`。

## 六、当前暂未自动接入的来源

中国政府采购网、沪深北交易所和巨潮资讯很重要，但目前没有在本项目中确认到可直接匿名调用且适合稳定增量采集的正式接口。因此暂时不使用网页私有请求硬抓。

外交部和国防科工局虽然有官方 RSS 说明页，但 2026-07-22 实际试采集没有返回有效条目，所以没有启用。等官方 XML 恢复后再重新验证。
